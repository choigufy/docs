API 명세
Supabase REST API — Next.js에서 직접 호출 (supabase-js 클라이언트)
GET daily_logs 작업 타임라인 조회
supabase.from("daily_logs").select("*").order("date", { ascending: false }) // 반환: id, date, title, content, tags, created_at // 정렬: 날짜 최신순
GET projects 포트폴리오 목록 조회
supabase.from("projects").select("*").order("date", { ascending: false }) // 반환: id, title, description, tech_stack, github_url, demo_url, thumbnail_url, date
GET blog_posts 공개된 블로그 포스트 조회
supabase.from("blog_posts").select("*").eq("published", true).order("created_at", { ascending: false }) // 필터: published=true 만 반환
GET claude_usage 토큰 사용량 전체 조회
supabase.from("claude_usage").select("*").order("date", { ascending: false }) // 반환: 날짜별 input/output/cache 토큰 + total_cost
UPSERT daily_logs / claude_usage Python 스크립트에서 데이터 저장
# Python (obsidian_sync.py, token_collector.py) supabase.table("daily_logs").upsert(data, on_conflict="date").execute() # on_conflict="date": 같은 날짜면 UPDATE, 없으면 INSERT
자동화 시스템
Python 스크립트 2종 — Windows 작업 스케줄러로 자동 실행
obsidian_sync.py — Obsidian 연동
1
Obsidian에서 Ctrl+S 저장
Day Planner-20260409.md
2
watchdog FileSystemEventHandler.on_modified() 감지
정규식 검증: ^Day Planner-\d{8}\.md$ | 중복 방지: 1초 이내 차단
3
parse_note() → 섹션 파싱
## 오늘 한 일 / ## 메모 / ## 내일 할 일 분리
4
extract_tags() → 태그 추출
정규식 r'#([^\s#]+)' + "daily" 기본 추가
5
Supabase daily_logs upsert
on_conflict="date" — 같은 날짜 수정 시 덮어쓰기
token_collector.py — 토큰 수집
1
작업 스케줄러 실행 (매일 23:50 + 로그인 시)
ClaudeTokenCollector 태스크
2
glob으로 JSONL 전체 탐색
~/.claude/projects/**/*.jsonl
3
type=="assistant" + usage 추출
input / output / cache_creation / cache_read 토큰
4
calc_cost() — 비용 계산
input×$3 + output×$15 + cache_c×$3.75 + cache_r×$0.30 (per 1M)
5
Supabase claude_usage upsert
on_conflict="date" — 매일 누적 갱신
